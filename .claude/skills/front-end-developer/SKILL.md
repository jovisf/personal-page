# Front-End Developer

## Instructions

When working on front-end tasks:

1. **Component Development**: Create reusable, semantic HTML components
2. **Styling**: Use CSS best practices including flexbox, grid, and responsive design
3. **JavaScript/TypeScript**: Write clean, efficient code following modern standards
4. **Debugging**: Use browser DevTools insights to troubleshoot issues
5. **Performance**: Optimize bundle size, minimize re-renders, lazy-load assets
6. **Accessibility**: Ensure WCAG compliance, semantic HTML, proper ARIA attributes
7. **Testing**: Help write unit and integration tests for front-end code

## Examples

**Component Creation**:
- Build React functional components with hooks
- Create Vue single-file components
- Develop vanilla JS modules

**Styling Tasks**:
- Convert designs to responsive CSS
- Debug layout issues
- Implement dark mode support

**Performance Optimization**:
- Code splitting and lazy loading
- Image optimization
- Bundle analysis

## Best Practices

- Use semantic HTML elements
- Follow component composition patterns
- Implement responsive design mobile-first
- Optimize assets and images
- Write accessible markup (ARIA labels, keyboard navigation)
- Use CSS variables for theming
- Minimize JavaScript bundle size
- Implement proper error boundaries

NEVER use generic AI-generated aesthetics like overused font families (Inter, Roboto, Arial, system fonts), cliched color schemes (particularly purple gradients on white backgrounds), predictable layouts and component patterns, and cookie-cutter design that lacks context-specific character.
